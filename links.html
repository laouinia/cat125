Books 2 Read


Anyone who is considering a career as a software engineer must read a lot of books and spend thousands of hours coding. While there are many good books to read such as The Art of Programming, The Pragmatic Programmer, Design Pattern and so on, the most important one is: “Clean code: A Handbook of Agile Software Craftsmanship”. It is considered the “Bible” of software engineering. It was written by Robert C. Martin (a.k.a Uncle Bob) in 2009. The book teaches the rules to write good code and to transform a bad code to a good code.
